# Dark Mode

Date: 2026-04-19

## What was added

- a root theme bootstrap script in `src/app/layout.tsx`
- a reusable floating toggle in `src/components/theme-toggle.tsx`
- persisted theme preference in `localStorage` under `manex-theme`
- shared dark-mode-ready color tokens for warning text, translucent elevated panels,
  and elevated overlays in `src/app/globals.css`

## Why the bootstrap script exists

Most screens are rendered through server components. If the app waits until React
hydrates before applying the `dark` class, users see a flash of the light theme on
first paint.

The inline script in `src/app/layout.tsx` resolves the stored preference, falls back
to `prefers-color-scheme`, and applies the `dark` class before the page paints.

## Current behavior

- first visit: follows the system color scheme
- after the user toggles theme: persists the explicit choice in `localStorage`
- the toggle is global because it is mounted in the root layout, not per page

## Notes for future UI work

Avoid hardcoding `text-amber-700`, `bg-white/78`, or similar light-only values in
feature components. Prefer theme tokens such as `--warning-foreground` and
`--raised-overlay-surface` so both themes stay readable.

The app-level page background is intentionally flat now. Do not add global
background gradients back into `body` for theme styling.
